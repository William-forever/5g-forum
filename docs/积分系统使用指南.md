# 积分系统使用指南

## 📊 系统概述

本积分系统为论坛用户提供积分激励机制，根据用户的活动行为自动计算积分，并在右侧边栏显示实时排行榜。

## 🎯 积分规则

| 行为 | 积分 | 说明 |
|------|------|------|
| 发布帖子 | +10分 | 每发布一个帖子 |
| 帖子被点赞 | +2分 | 自己的帖子被他人点赞一次 |
| 帖子被收藏 | +2分 | 自己的帖子被他人收藏一次 |
| 发布评论 | +1分 | 每发布一条评论 |
| 点赞他人 | +0.5分 | 点赞他人的帖子或评论 |
| 收藏他人 | +0.5分 | 收藏他人的帖子 |

## 🏆 排行榜显示

积分排行榜显示在论坛右侧边栏，包含以下功能：

- **实时更新**：每小时自动更新一次
- **手动刷新**：点击刷新按钮可立即更新
- **时间显示**：显示上次更新和下次更新时间
- **用户头像**：显示用户头像或首字母图标
- **前3名标识**：前3名用户有特殊标识

## ⚙️ 系统配置

### 环境变量

```bash
# 定时任务API密钥（可选）
CRON_API_KEY=your-secret-key

# 启用定时任务（默认启用）
ENABLE_CRON=true

# 测试模式（每分钟执行一次）
CRON_TEST_MODE=false
```

### 数据库表

系统使用以下数据库表：

1. **users** - 用户表（已有）
2. **posts** - 帖子表（已有）
3. **comments** - 评论表（已有）
4. **likes** - 点赞表（新增）
5. **collections** - 收藏表（新增）
6. **points_log** - 积分日志表（新增）

## 🚀 快速开始

### 1. 启动服务

```bash
# 开发模式
npm run dev

# 生产模式
npm run build
npm start
```

### 2. 初始化数据库

确保数据库中有以下表结构：

```sql
-- 点赞表
CREATE TABLE IF NOT EXISTS likes (
  id SERIAL PRIMARY KEY,
  user_id INTEGER NOT NULL,
  target_id INTEGER NOT NULL,
  target_type INTEGER NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(user_id, target_id, target_type)
);

-- 收藏表
CREATE TABLE IF NOT EXISTS collections (
  id SERIAL PRIMARY KEY,
  user_id INTEGER NOT NULL,
  post_id INTEGER NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(user_id, post_id)
);

-- 积分日志表
CREATE TABLE IF NOT EXISTS points_log (
  id SERIAL PRIMARY KEY,
  user_id INTEGER NOT NULL,
  points_change INTEGER NOT NULL,
  total_points INTEGER NOT NULL,
  reason VARCHAR(200),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 3. 手动更新排行榜

```bash
# 一次性更新
npm run update-rankings

# 启动定时任务
npm run cron

# 测试模式（每分钟执行一次）
npm run cron:test
```

## 🔧 API接口

### 获取排行榜

```http
GET /api/users/rankings?limit=10
```

参数：
- `limit`：返回的用户数量（默认10）
- `force_update`：是否强制更新（true/false）

响应：
```json
{
  "success": true,
  "data": [
    {
      "rank": 1,
      "username": "张三",
      "points": 1280,
      "avatar": "https://...",
      "isTop3": true
    }
  ],
  "meta": {
    "count": 10,
    "timestamp": "2026-03-13T10:30:00Z",
    "next_update": "2026-03-13T11:30:00Z"
  }
}
```

### 手动更新排行榜

```http
POST /api/cron/update-rankings
```

请求头：
```http
X-API-Key: your-secret-key
```

## 🛠️ 故障排除

### 常见问题

1. **排行榜不显示**
   - 检查数据库连接
   - 检查API接口是否正常
   - 查看浏览器控制台错误

2. **积分计算不准确**
   - 检查数据库表结构
   - 验证积分计算逻辑
   - 检查数据完整性

3. **定时任务不执行**
   - 检查环境变量 `ENABLE_CRON`
   - 查看服务器日志
   - 验证时区设置

### 日志查看

```bash
# 查看Next.js日志
npm run dev 2>&1 | grep "CRON"

# 查看定时任务日志
npm run cron:test
```

## 📈 性能优化

### 数据库索引

建议添加以下索引以提高查询性能：

```sql
-- 用户积分索引
CREATE INDEX idx_users_points ON users(points DESC);

-- 点赞用户索引
CREATE INDEX idx_likes_user_id ON likes(user_id);
CREATE INDEX idx_likes_target ON likes(target_id, target_type);

-- 收藏用户索引
CREATE INDEX idx_collections_user_id ON collections(user_id);

-- 积分日志索引
CREATE INDEX idx_points_log_user_id ON points_log(user_id);
CREATE INDEX idx_points_log_created_at ON points_log(created_at DESC);
```

### 缓存策略

1. **排行榜缓存**：缓存5分钟，减少数据库查询
2. **用户积分缓存**：缓存用户最新积分
3. **页面缓存**：静态页面缓存
